REST Framework

A lightweight REST framework backed by an encrypted JSON document database.

Authentication

Login

POST /login

{
    "username": "admin",
    "password": "password"
}
{
    "success": true,
    "data": {
        "token": "<jwt-token>",
        "refresh_token": "<refresh-token>",
        "expires_in": 3600
    }
}

Returns a short-lived JWT access token and a rotating refresh token.

Refresh Token

POST /refresh

{
    "refresh_token": "<refresh-token>"
}
{
    "success": true,
    "data": {
        "token": "<jwt-token>",
        "refresh_token": "<refresh-token>",
        "expires_in": 3600
    }
}

Returns a new JWT access token and a new refresh token. Refresh tokens are rotated on every successful refresh.

Logout

POST /logout

Revokes the current session and all associated refresh tokens.

Current User

GET /whoami

Authorization: Bearer <jwt-token>

Users

  • GET /api/users
  • GET /api/user/{id}
  • POST /api/user
  • PUT /api/user/{id}
  • PUT /api/user/password
  • DELETE /api/user/{id}

Sessions

  • GET /api/sessions
  • GET /api/session/{id}
  • PUT /api/session/{id}
  • DELETE /api/session/{id}

Sessions represent authenticated devices or browsers. Each session maintains its own rotating refresh token chain and can be revoked independently.

Items

  • GET /api/items
  • GET /api/item/{id}
  • POST /api/item
  • PUT /api/item/{id}
  • DELETE /api/item/{id}

Logs

  • GET /logs
  • GET /log/{id}
  • DELETE /log/{id}

Filtering

All filters are passed as query parameters. String comparisons are case-insensitive. Dates uses the ISO 8601 UTC format with a trailing Z.

Operator Syntax Example
Equals field=value ?name=John
Not Equals field[not]=value ?name[not]=John
Greater Than field[gt]=value ?price[gt]=100
Greater Than or Equal field[ge]=value ?price[ge]=100
Less Than field[lt]=value ?price[lt]=100
Less Than or Equal field[le]=value ?price[le]=100
Contains field[contains]=value ?name[contains]=Lap
Starts With field[startswith]=value ?name[startswith]=Lap
Ends With field[endswith]=value ?name[endswith]=top
In field=v1,v2,v3 ?category=electronics,computers
Nested Object parent[child]=value ?customer[city]=London
Array Contains field=[value] ?role=[admin]
Date Comparison field[gt]=yyyy-mm-ddThh:mm:ssZ ?created_at=2026-06-30T08:40:53Z

Examples

?name=Laptop

?price[gt]=100

?price[gt]=100

?price[ge]=100

?price[lt]=1000

?price[le]=1000

?name[contains]=lap

?name[startswith]=lap

?name[endswith]=top

?category=electronics,computers

?customer[city]=London

?tags=[electronics]

?created_at[gt]=2026-06-30T08:40:53Z

?created_at[ge]=2026-06-30T08:40:53Z

?updated_at[lt]=2026-07-01T00:00:00Z

Note: Date filters should use UTC format YYYY-MM-DDTHH-MM-SSZ. Using Z avoids URL encoding issues with timezone offsets such as +00:00.

Response Format

Success

{
	"success": true,
	"data": [
		...
	]
}

Successful responses may return different HTTP status codes depending on the operation (for example 200 OK, 201 Created or 204 No Content).

Error

{
	"success": false,
	"error": "..."
}
